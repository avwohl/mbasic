    Implementation Notes¶

    Documentation Template¶

    Each function/statement doc should have: - Title and syntax - Versions where available (SK, Extended, Disk Extended, Disk) - Clear description - Working example(s) - Notes section (edge cases, gotchas) - See Also section with links

    Naming Conventions¶

    Decision needed: Should we use STR or STR$ in docs? - MBASIC 5.21 prints functions with $ suffix - Implementation uses _DOLLAR suffix internally - Docs currently inconsistent

    Recommendation: Use STR$ in docs (user-visible), STR in implementation
