ISSUES FOUND (String DATA):
================================================================================

Issue: Real MBASIC doesn't support unquoted string literals in DATA
----------------------------------------------------------------------

Test case that FAILS in real MBASIC:
  220 DATA ALICE, NYC

Error from real MBASIC:
  Syntax error in 220

Conclusion: String DATA must either be:
1. Quoted: DATA "ALICE", "NYC" (but this also causes syntax error in real MBASIC!)
2. Or strings might not be supported in DATA at all in MBASIC 5.21

Note: Further testing needed to determine if string DATA is supported in
MBASIC 5.21. The original games use DATA extensively but mostly with numbers.
